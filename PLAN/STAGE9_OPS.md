# 阶段9: 运维管理系统 — 产品设计方案

## 概述

为 Text-to-SQL + RAG 融合项目构建管理后台，实现表权限管理、文档权限管理、运维操作 CRUD。

## 管理后台布局

```
聊天页(/)                 管理后台(/admin)
┌──────────────┐         ┌──────┬─────────────────┐
│ 侧边栏 │ 聊天区 │         │ 导航  │ 表格/表单/详情   │
│        │       │   →    │      │                 │
└──────────────┘         │ ←返回 │                 │
                          └──────┴─────────────────┘
```

侧边栏导航：数据表管理 | 文档管理 | 权限总览 | ← 返回聊天

---

## 功能一：数据库表管理

### 表列表
- 搜索+筛选+表格（表名/注释/预算类型/列数/地区权限数/状态开关/操作）
- 新增表：输入表名→自动从达梦同步结构→默认启用
- 编辑：修改注释、预算类型标签
- 删除：软删除（禁用），不物理删除

### 表状态
- 禁用后 list_tables 和 search_schema 不再返回此表
- Switch 开关即时切换

---

## 功能二：地区权限管理

### 权限模型
表↔地区 多对多关系，参考 `region_tree.py` 9位行政区划编码。

| 设置 | 生效范围 | 规则 |
|------|---------|------|
| 河北省 130000000 | 全省 | 组节点→前缀 LIKE '13%' |
| 石家庄 130100000 | 本市+区县 | 组节点→前缀 LIKE '1301%' |
| 长安区 130102000 | 仅本区 | 叶子节点→精确匹配 |

### 权限生效
- `run_sql` 执行前自动在 WHERE 中注入 RG_CODE 过滤
- 用户已写 RG_CODE 条件时不覆盖用户意图

### 权限配置界面
- 级联选择器（省/市/区县）+ 快捷操作按钮
- 表权限和文档权限复用同一组件

---

## 功能三：文档管理

### 文档列表
- 卡片布局（文件名/来源/向量数/状态/操作）
- 上传文档：拖拽上传→自动向量化→写入 pgvector→状态=已启用
- 删除：移除向量+标记删除，保留原始文件
- 启用/禁用：Switch 控制 RAG 检索是否包含

---

## 功能四：权限总览

按地区查看可访问的表和文档列表，发现权限覆盖缺口。

---

## 元数据存储：SQLite

`data/admin.db`，5张表：
- `table_metadata`：表名/注释/预算类型/列数/状态
- `table_regions`：表↔地区
- `document_metadata`：文件名/路径/向量数/状态
- `document_regions`：文档↔地区
- `audit_log`：操作日志

---

## API 端点（/api/admin/）

| 类别 | 端点 |
|------|------|
| 表管理 | GET/POST /tables, GET/PUT/DELETE /tables/{name}, PUT toggle, POST sync |
| 表权限 | GET/PUT /tables/{name}/regions |
| 文档管理 | GET/POST /documents, DELETE /documents/{source}, PUT toggle |
| 文档权限 | GET/PUT /documents/{source}/regions |
| 权限总览 | GET /overview?region_code=xxx |
| 地区树 | GET /regions/tree |
| 日志 | GET /logs |
