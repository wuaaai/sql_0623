## 一、用户角色与使用场景

### 1.1 角色定义

| 角色 | 职责 | 使用频率 |
|------|------|---------|
| 系统管理员 | 管理全部表和文档，配置地区权限 | 每周1-2次 |
| 业务管理员 | 管理自己负责地区的表权限和文档 | 每月数次 |
| 普通用户 | 只使用聊天查询，不进入管理后台 | 每天 |

### 1.2 核心使用场景

**场景A: 新表上线**
> 达梦数据库新增了一张预算执行表，管理员需要把它加入到系统中，并限制只有石家庄市的用户可以查询。

操作流程：
1. 管理员打开管理后台 → 数据表管理
2. 点击"新增表"，输入表名 `RDYS_LD_YSZX_NEW_TABLE`
3. 系统自动从达梦获取列结构（45列），默认启用、不限地区
4. 管理员点击该表的"权限"按钮 → 勾选"石家庄市(130100000)" → 保存
5. 返回聊天页，用户查询"石家庄市预算收入" → 自动包含新表数据
6. 用户查询"唐山市预算收入" → 不包含此表

**场景B: 文档权限调整**
> 邯郸市的预算解读文档之前设置为全省可见，现在需要改为仅邯郸市可见。

操作流程：
1. 管理员进入文档管理 → 找到"2026年邯郸市预算解读.docx"
2. 点击"权限" → 当前显示"河北省(130000000)"
3. 移除"河北省" → 添加"邯郸市(130400000)" → 保存
4. 石家庄用户问"预算编制要求" → 不再返回邯郸文档内容

**场景C: 临时禁用表**
> 某张表的数据正在更新维护中，需要暂时对用户隐藏。

操作流程：
1. 管理员在表列表中找到该表
2. 点击状态 Switch → 切换为"禁用"
3. 用户查询时 `list_tables` 和 `search_schema` 不再返回此表
4. 维护完成后，管理员再切换为"启用"

**场景D: 权限覆盖检查**
> 管理员想知道"唐山市的用户能看到哪些表和文档"。

操作流程：
1. 进入权限总览页
2. 选择地区"唐山市(130200000)"
3. 系统展示：可访问24张表、5个文档
4. 发现缺少某张重要的表 → 点击"去配置" → 直接跳转配置

- "清空所有权限" → 移除所有记录 → 恢复默认（无限制）

**权限继承规则**（SQL 注入用）:

```

设置 130100000(石家庄) → WHERE RG_CODE LIKE '1301%'  
设置 130102000(长安区) → WHERE RG_CODE = '130102000'
设置 [130100000, 130200000] → WHERE (RG_CODE LIKE '1301%' OR RG_CODE LIKE '1302%')
空权限 → 不注入任何限制
```

### 4.2 权限在 run_sql 中的注入逻辑

```python
# 伪代码: handler.py do_run_sql 中的权限检查
def do_run_sql(self, args):
    sql = args["sql"]
    table_name = _extract_table_from_sql(sql)  # 从 FROM 提取表名
    
    # 1. 查表是否启用
    meta = admin_db.get_table_metadata(table_name)
    if meta and not meta.is_enabled:
        return {"status": "error", "message": "此表已被管理员禁用"}
    
    # 2. 查表是否有地区权限限制
    regions = admin_db.get_table_regions(table_name)
    if regions and not _sql_has_rg_condition(sql):
        # 用户SQL中没有RG_CODE条件 → 注入权限过滤
        region_clause = _build_region_where(regions)
        sql = _inject_where(sql, region_clause)
    
    # 3. 执行SQL
    return db_query.run_sql(sql)
```

```sql
CREATE TABLE table_metadata (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    table_name TEXT NOT NULL UNIQUE,
    comment TEXT DEFAULT '',
    budget_type TEXT DEFAULT '其他',
    column_count INTEGER DEFAULT 0,
    is_enabled INTEGER DEFAULT 1,
    columns_json TEXT DEFAULT '[]',
    created_at TEXT NOT NULL DEFAULT (datetime('now','localtime')),
    updated_at TEXT NOT NULL DEFAULT (datetime('now','localtime'))
);

-- 表地区权限
CREATE TABLE table_regions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    table_name TEXT NOT NULL,
    region_code TEXT NOT NULL,
    created_at TEXT NOT NULL DEFAULT (datetime('now','localtime')),
    UNIQUE(table_name, region_code)
);

-- 文档元数据
CREATE TABLE document_metadata (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    source TEXT NOT NULL UNIQUE,
    file_path TEXT NOT NULL,
    chunk_count INTEGER DEFAULT 0,
    is_enabled INTEGER DEFAULT 1,
    status TEXT DEFAULT 'enabled',
    created_at TEXT NOT NULL DEFAULT (datetime('now','localtime')),
    updated_at TEXT NOT NULL DEFAULT (datetime('now','localtime'))
);

-- 文档地区权限
CREATE TABLE document_regions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    source TEXT NOT NULL,
    region_code TEXT NOT NULL,
    created_at TEXT NOT NULL DEFAULT (datetime('now','localtime')),
    UNIQUE(source, region_code)
);

-- 操作日志
CREATE TABLE audit_log (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    action TEXT NOT NULL,
    target_type TEXT NOT NULL,
    target_name TEXT NOT NULL,
    detail TEXT DEFAULT '',
    created_at TEXT NOT NULL DEFAULT (datetime('now','localtime'))
);
```

### 7.3 首次初始化

服务启动时执行：
1. 检查 `data/admin.db` 是否存在
2. 不存在 → 创建库 + 建表 + 从达梦批量导入47张表（调用 `DESCRIBE TABLE`）
3. 从 pgvector 按 `source` 去重导入文档记录
4. 全部默认启用、不限地区

## 九、API 端点详细设计

### 9.1 表管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/admin/tables` | 表列表，支持 `?search=xxx&budget_type=xxx&page=1&page_size=20` |
| GET | `/api/admin/tables/{table_name}` | 表详情（含列结构） |
| POST | `/api/admin/tables` | 新增表 `{table_name, comment, budget_type}` |
| PUT | `/api/admin/tables/{table_name}` | 编辑 `{comment, budget_type}` |
| DELETE | `/api/admin/tables/{table_name}` | 软删除（is_enabled=0） |
| PUT | `/api/admin/tables/{table_name}/toggle` | 切换启用/禁用 |
| POST | `/api/admin/tables/{table_name}/sync` | 重新同步列结构 |

### 9.2 表权限

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/admin/tables/{table_name}/regions` | 获取权限列表 `{regions: ["130000000", "130100000"]}` |
| PUT | `/api/admin/tables/{table_name}/regions` | 全量替换权限 `{regions: ["130000000"]}` |

### 9.3 文档管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/admin/documents` | 文档列表，支持 `?search=xxx&page=1&page_size=12` |
| POST | `/api/admin/documents` | multipart上传：`file` + `regions` JSON字段 |
| DELETE | `/api/admin/documents/{source}` | 删除文档（移除向量+标记） |
| PUT | `/api/admin/documents/{source}/toggle` | 切换启用/禁用 |

### 9.4 文档权限 + 总览 + 地区树

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/admin/documents/{source}/regions` | 获取文档权限 |
| PUT | `/api/admin/documents/{source}/regions` | 全量替换 |
| GET | `/api/admin/overview?region_code=130100000` | 权限总览 |
| GET | `/api/admin/regions/tree` | 地区编码树 |
| GET | `/api/admin/logs?page=1&page_size=50` | 操作日志 |
