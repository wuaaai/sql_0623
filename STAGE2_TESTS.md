# 阶段2 验收测试问题集 — 预算业务精准查询

> 设计原则：所有问题均为预算分析人员真实口语，不含任何英文表名、列名、编码。
> Agent 必须自动完成口语→表/列/值的映射。用户只描述业务需求。

---

## 一、预算类型识别 — "我要查哪类预算"

用户只描述预算类别，Agent 自动定位到正确的表。

| # | 用户问题 | Agent 必须做的事 |
|---|---------|----------------|
| 1 | 一般公共预算收入完成得怎么样 | search_schema("一般公共预算收入") → 自动扩展到 YBGGYS → 找到收入表 → describe → run_sql |
| 2 | 社保基金的支出情况帮我看看 | search_schema("社保基金支出") → SHBXJJ → 支出表 |
| 3 | 国有资本经营预算的情况 | search_schema("国有资本") → GYZBJY |
| 4 | 政府性基金收入怎么样 | search_schema("政府性基金") → ZFXJJ |
| 5 | 一般预算里面税收收入完成多少 | 找到一般公共预算收入表 → 项目筛"税收" |
| 6 | 社保相关的表都有哪些 | search_schema("社保") → 列出 SHBXJJ 的几张表，用中文说明每张表用途 |
| 7 | 国资预算收入完成得咋样 | 理解"国资"=国有资本经营预算 → GYZBJY 收入表 |
| 8 | 四本预算里关于基金的那个 | 区分"政府性基金"和"社保基金"，列出候选让系统自动判断或追问 |

## 二、地区筛选 — "我要看哪里的"

用户只说地区名，Agent 自动映射到正确的列和值。

| # | 用户问题 | Agent 必须生成的 WHERE 条件 |
|---|---------|--------------------------|
| 9 | 只看河北省的 | `WHERE RG_NAME = '河北省'` |
| 10 | 石家庄的情况 | `WHERE RG_NAME LIKE '%石家庄%'`（数据库可能存"石家庄市"） |
| 11 | 省本级的数据 | `WHERE RG_NAME LIKE '%本级%'` |
| 12 | 各地市的，不要省本级 | `WHERE RG_NAME NOT LIKE '%本级%'` |
| 13 | 石家庄和保定两个地方 | `WHERE RG_NAME LIKE '%石家庄%' OR RG_NAME LIKE '%保定%'` |
| 14 | 河北各市排名 | NOT LIKE '%本级%' + ORDER BY 收入列 DESC |
| 15 | 唐山的 | `WHERE RG_NAME LIKE '%唐山%'` |
| 16 | 开发区那些 | `WHERE RG_NAME LIKE '%开发区%'` 或 `LIKE '%高新%'` |

## 三、时间筛选 — "我要看什么时候的"

用户只说口头时间，Agent 推断正确的 YEAR_MONTH 格式。

| # | 用户问题 | Agent 必须生成的 WHERE 条件 |
|---|---------|--------------------------|
| 17 | 2025年1月的 | `WHERE YEAR_MONTH = '202501'` |
| 18 | 今年的 | 推断当前年份2026 → `WHERE YEAR_MONTH LIKE '2026%'` |
| 19 | 最近三个月 | 计算最近三个 YEAR_MONTH → `WHERE YEAR_MONTH IN (...)` |
| 20 | 2025年全年的 | `WHERE YEAR_MONTH LIKE '2025%'` |
| 21 | 和去年同期比一比 | 选同比列：BYS_SNTYS, BYS_TBE, BYS_TBB |
| 22 | 2020年到2025年这几年 | `WHERE YEAR_MONTH >= '202001' AND YEAR_MONTH <= '202512'` |

## 四、项目筛选 — "我要看什么类型的收支"

用户只说业务术语，Agent 映射到 XM_NAME 条件。

| # | 用户问题 | Agent 必须生成的 WHERE 条件 |
|---|---------|--------------------------|
| 23 | 只看合计 | `WHERE XM_NAME LIKE '%合计%'` |
| 24 | 税收收入完成多少 | `WHERE XM_NAME LIKE '%税收%' AND XM_NAME NOT LIKE '%非税%'` |
| 25 | 非税收入的情况 | `WHERE XM_NAME LIKE '%非税%'` |
| 26 | 合计、税收、非税都列出来 | WHERE 三个条件的 OR 组合 |
| 27 | 专项收入 | `WHERE XM_NAME LIKE '%专项%'` |
| 28 | 除了合计还有什么 | SELECT DISTINCT XM_NAME WHERE NOT LIKE '%合计%' |
| 29 | 行政事业性收费的那个 | `WHERE XM_NAME LIKE '%行政%'` |

## 五、同比对比 — "和去年比怎么样"（核心场景）

用户问"增长/下降/比去年"，Agent 自动选同比列。

| # | 用户问题 | Agent 必须做的事 |
|---|---------|----------------|
| 30 | 河北1月收入比去年涨了还是跌了 | 选 BYS_JE, BYS_SNTYS, BYS_TBE, BYS_TBB 四列 |
| 31 | 哪些地区收入同比下降了 | `WHERE BYS_TBE < 0` + ORDER BY |
| 32 | 社保基金收入同比增长超过百分之十的地区 | `WHERE BYS_TBB > 10` |
| 33 | 和去年比多了多少 | 选同比增减额和同比增减率两列 |
| 34 | 国有资本预算收入和去年同期对比看看 | 定位 GYZBJY 收入表 → 选同比列 |
| 35 | 政府性基金支出同比降幅最大的是哪个地区 | `WHERE BYS_TBE < 0 ORDER BY BYS_TBE ASC` + ROWNUM <= 1 |
| 36 | 这个月收入和上个月比 | 识别用户要的是"环比"而非"同比"，提示目前支持同比对比 |
| 37 | 收入负增长的有哪些 | 理解"负增长"=同比下降 → `WHERE BYS_TBB < 0` |

## 六、组合查询 — "多个条件一起说"

用户一口气说完所有条件，Agent 自动拆解并组合。

| # | 用户问题 | Agent 必须组合的条件 |
|---|---------|-------------------|
| 38 | 河北省2025年1月一般公共预算收入合计是多少 | 地区=河北省 + 时间=202501 + 预算=一般公共预算收入 + 项目=合计 → 4个条件 |
| 39 | 石家庄社保基金本月收入同比怎么样 | 地区=石家庄 + 预算=社保 + 时间=本月 + 同比列 → 4个条件 |
| 40 | 各市2025年政府性基金累计收入，排除省本级 | 时间=2025 + 预算=政府性基金 + 排除本级 + 累计列 |
| 41 | 保定廊坊2025上半年一般公共预算支出合计 | 多地区 + 时间范围(202501-202506) + 预算类型 + 项目 |
| 42 | 2025年各市国有资本预算收入合计，看看同比 | 时间=2025 + 预算=国资 + 排除本级 + 项目=合计 + 同比列 |
| 43 | 河北省一般公共预算的税收和非税分别完成多少 | 地区=河北 + 预算=一般公共预算 + 两项目筛选 |

| # | 用户问题 | Agent 应选的列（非 SELECT *） |
|---|---------|----------------------------|
| 44 | 一般公共预算收入的完成情况 | 地区、时间、项目名、预算数、本月金额、累计金额 |
| 45 | 社保基金的同比情况 | 地区、时间、项目名、本月金额、上年同期、增减额、增减率 |
| 46 | 预算执行进度怎么样 | 地区、项目名、预算数、本月完成、累计完成 |

## 八、容错与边界 — "用户说错了或说不清"

| # | 用户问题 | Agent 必须做的事 |
|---|---------|----------------|
| 51 | 河北省13月的数据 | 提示月份无效（01-12），不是生成 WHERE YEAR_MONTH = '202513' |
| 52 | 火星省的收入 | 返回0行 → 友好提示"未找到火星省，可能是名称不匹配" |
| 53 | 一般公共预算收入有多少 | 追问：哪个地区、哪个时间段？不能静默查全库 |
| 54 | 帮我分析一下预算情况 | 追问：哪种预算类型？不能自己猜一张表就查 |
| 55 | 石家庒的预算收入 | 容错 LIKE '%石家%'（用户打了错字"庒"） |
| 56 | 一般公共预算和政府性基金哪个增长快 | 跨预算对比 → 告知当前阶段分别查询，后续版本支持 |

---

## 验收标准

- **P0（1-50）**：≥90% 通过（至少45题行为正确）
- **SQL质量**：关键字大写、ROWNUM语法正确、不编造列名
- **列选择**：除非用户明确要求，不使用 SELECT *
- **用户体验**：所有返回结果用中文解释，英文列名保留但附带说明

## 九、复杂场景 — 多步推理与分析

这些场景需要 Agent 综合运用多个技能。

| # | 用户问题 | 涉及技能 | Agent 必须做的事 |
|---|---------|---------|----------------|
| 57 | 河北省一般公共预算收入完成进度怎么样，和全年预算比完成了多少 | 预算执行 + 同比 | 选 YSS, BYS_JE, BYLJS_JE → 计算完成率 → 输出百分比 |
| 58 | 哪些地市预算执行偏慢，帮我标出来 | 预算执行 + 排名 | 各市完成率 → ORDER BY ASC → 对 <50% 的标注"偏慢" |
| 59 | 有没有超预算的项目 | 预算执行 | WHERE BYLJS_JE > YSS → 计算超支额和比例 |
| 60 | 社保基金收入同比增长最快的前5个地区，也看看它们的支出情况 | 同比 + 多表 | 先查收入同比排名 → 再查对应的支出表 |
| 61 | 今年各月一般公共预算收入的走势，有没有哪个月特别异常 | 时间趋势 + 异常检测 | GROUP BY YEAR_MONTH → 观察各月波动 → 标注偏离均值较大的月份 |
| 62 | 政府性基金和国有资本，哪个的预算执行进度更好 | 跨类型对比 | 分别查两个预算类型的完成率 → 对比 → 给出结论 |
