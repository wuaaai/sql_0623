# 阶段2 产品计划 — 条件筛选（预算业务场景）

## 业务背景

用户是财政预算分析人员，日常工作围绕四大预算类型：
- **一般公共预算** (YBGGYS) — 最常用，7张表
- **社会保险基金预算** (SHBXJJYS) — 7张表
- **国有资本经营预算** (GYZBJYYS) — 7张表
- **政府性基金预算** (ZFXJJ) — 4张表

用户口语化程度高，不会说表名或列名，只会用业务术语描述需求。

## 数据特征

所有预算表共用核心字段结构：

| 字段 | 含义 | 用户怎么说 |
|------|------|-----------|
| `RG_NAME` | 地区名称 | "河北省"、"石家庄" |
| `YEAR_MONTH` | 年月 | "1月份"、"2020年" |
| `XM_NAME` | 项目名称 | "合计"、"税收收入" |
| `YSS` | 预算数 | "全年预算"、"预算安排" |
| `BYS_JE` / `BY_JE` | 本月金额 | "完成多少"、"收入多少" |
| `BYS_SNTYS` / `BY_SNTYS` | 上年同期 | "去年同期" |
| `BYS_TBE` / `BY_TBE` | 同比增减额 | "比去年多了多少" |
| `BYS_TBB` / `BY_TBBFS` | 同比增减率% | "增长了多少百分比" |
| `BYLJS_JE` | 本月累计金额 | "今年累计"、"1月到现在" |

## 产品功能拆解

### 功能2.1 按预算类型查找表

用户不会说表名，只知道预算类型。Agent 必须自动完成映射。

**典型问法：**
- "一般公共预算收入完成情况" → search_schema → 找 YBGGYS + 收入 + 完成
- "社保基金支出" → search_schema → 找 SHBXJJ + 支出
- "国有资本经营预算" → search_schema → 找 GYZBJY
- "政府性基金收入" → search_schema → 找 ZFXJJ + 收入

### 功能2.2 按地区筛选

**典型问法：**
- "河北省的数据" → WHERE RG_NAME = '河北省'
- "石家庄的" → WHERE RG_NAME LIKE '%石家庄%'
- "省本级" → WHERE RG_NAME LIKE '%本级%'
- "各市的情况" → WHERE RG_NAME NOT LIKE '%本级%'（排除本级，只看地市）

### 功能2.3 按时间筛选

**典型问法：**
- "1月份" → WHERE YEAR_MONTH LIKE '%01' 或 = '202001'
- "今年" / "2025年" → WHERE YEAR_MONTH LIKE '2025%'
- "最近三个月" → 可能需要多个 YEAR_MONTH 值
- "去年同期" → 计算上一年同月

### 功能2.4 按项目筛选

**典型问法：**
- "合计" → WHERE XM_NAME LIKE '%合计%'
- "税收收入" → WHERE XM_NAME LIKE '%税收%'
- "非税收入" → WHERE XM_NAME LIKE '%非税%'

### 功能2.5 同比对比查询（核心场景）

这是用户最常问的模式，涉及同比增减额(TBE)和同比增减率(TBB)。

**典型问法：**
- "河北1月收入比去年多了还是少了" → 查 BYS_TBE / BYS_TBB
- "石家庄社保收入同比增了多少" → 查同比字段
- "哪些地区收入同比下降了" → WHERE BYS_TBE < 0
- "国有资本预算同比增长超过10%的地区" → WHERE BYS_TBB > 10

### 功能2.6 条件组合查询

**典型问法：**
- "河北省2025年1月一般公共预算收入合计" → WHERE RG_NAME='河北省' AND YEAR_MONTH='202501' AND XM_NAME LIKE '%合计%'
- "石家庄市社保基金本月收入同比情况" → 多条件 + 选择同比列
- "2025年各市政府性基金累计收入" → 时间 + 累计列 + 排除本级

## 分步实现计划

### 第1步: 预算类型 → 表名映射 (核心基础)

Agent 必须能从"一般公共预算"找到对应的7张表。

**实现方式：** 全局记忆中存储四类预算→表名前缀的映射关系，Agent 结合 search_schema 精确定位。

### 第2步: 单条件筛选 — 地区

用户问"XX地区的数据"，Agent 能生成正确的 WHERE RG_NAME = 'XX'。

**验收用例：**
- "查看河北省的数据"
- "石家庄市的预算收入"
- "省本级的情况"
- "排除省本级，看各地市"

### 第3步: 单条件筛选 — 时间

用户问"X月/XX年"，Agent 理解 YEAR_MONTH 格式。

**验收用例：**
- "2025年1月的数据"
- "今年的数据"
- "最近一个月"

### 第4步: 单条件筛选 — 项目名称

用户问"合计/税收/非税"，Agent 筛选 XM_NAME。

**验收用例：**
- "只看合计"
- "税收收入的情况"
- "非税收入有哪些"

### 第5步: 多条件组合 + 按列筛选

用户问"河北省1月的一般公共预算收入合计"，Agent 组合 地区+时间+项目 三个条件，并且只选择与用户问题相关的列。

**验收用例：**
- "河北省2025年1月一般公共预算收入合计" → 3个条件 + 限定列
- "石家庄社保基金本月收入" → 2个条件
- "2025年各市政府性基金累计收入，排除省本级" → 时间+累计列+排除条件

### 第6步: 同比对比查询

用户问"增了多少/降了多少/比去年"，Agent 查同比字段(TBE/TBB)。

**验收用例：**
- "河北1月收入同比情况" → 显示 BYS_TBE / BYS_TBB
- "哪些地区收入同比下降了" → WHERE BYS_TBE < 0
- "国有资本预算同比增超10%的地区" → WHERE BYS_TBB > 10
- "社保基金收入和去年同期比怎么样" → 展示 BYS_JE + BYS_SNTYS + BYS_TBB

### 第7步: 相关列智能选择

用户描述关注点，Agent 只查相关列而不是 SELECT *。

**验收用例：**
- "看看收入完成情况" → 选 BYS_JE, BYLJS_JE, YSS
- "看看同比情况" → 选 BYS_JE, BYS_SNTYS, BYS_TBE, BYS_TBB
- "看看预算执行进度" → 选 YSS, BYS_JE, BYLJS_JE
