# Text-to-SQL 产品功能路线图

## 产品概述

**产品定位**: 面向财政预算分析人员的自然语言数据库查询助手

**目标用户**: 省/市级财政预算分析人员，熟悉预算业务但不懂 SQL 和数据库表结构

**核心价值**: 用户用口语化的业务术语描述需求 → 系统自动理解、定位数据、生成正确 SQL、返回结果

**当前数据环境**: 达梦数据库 · Schema: RDYS_PUBLIC_TBS · 47张表

---

## 用户画像

| 角色 | 特征 | 典型行为 |
|------|------|----------|
| 预算分析员 | 5年以上财政业务经验，不懂技术 | "看看这个月一般公共预算收入完成多少" |
| 科室负责人 | 关注汇总和排名，需要快速决策 | "哪个地市国有资本预算收入增长最快" |
| 上级领导 | 只看关键数字，要结果不要过程 | "今年社保基金收支总体情况" |

**用户语言特征**: 口语化、用业务别名、不区分表名/列名、习惯用"大概/差不多/好像"等模糊词

### 阶段1: 基础探索 — "能查了" ✅

**产品价值**: 用户第一次使用就能连接数据库、看到有哪些表、查询单张表的数据。

**用户故事**:
- 作为预算分析员，我想知道系统里有哪些数据表，以便了解可查询的数据范围
- 作为科室负责人，我想快速看一眼某张表有什么数据，以便确认数据是否最新
- 作为新用户，我描述的"支出相关的表"能被系统理解并找到正确的表

**核心功能**:

| 编号 | 功能 | 用户输入示例 | 系统行为 |
|------|------|-------------|----------|
| 1.1 | 数据库连接 | (前端点击连接) | 自动连接达梦数据库 |
| 1.2 | 列出所有表 | "数据库有哪些表" | 返回47张表名，按前缀分组 |
| 1.3 | 模糊搜索表 | "支出相关的表" | search_schema 搜索中文注释，返回匹配表 |
| 1.4 | 查看表结构 | "看看支出分类表的结构" | 返回列名/类型/样例数据 |
| 1.5 | 简单数据查询 | "查看前5行" | SELECT * WHERE ROWNUM <= 5 |

**验收标准**:
- 用户说"支出相关的表"，Agent 调用 search_schema("支出")，找到11张表
- 用户说"看看这张表的结构"，Agent 调用 describe_table，返回完整列信息
- 用户说"查前5行"，Agent 生成正确的 ROWNUM 语法
- 所有返回的表名/列名使用原始英文名，不编造中文含义

**已实现的原子工具**: connect_db, list_tables, search_schema, describe_table, run_sql

### 阶段2: 精准查询 — "能筛选了" 🔄

**产品价值**: 用户能用口语化的业务术语描述筛选条件，系统自动定位到正确的表和列，生成带 WHERE 条件的精确查询。

**核心理念**: 用户不知道表名叫 `RDYS_LD_YSZX_YBGGYS_DQZCWCQK`，他只知道"一般公共预算"。用户不知道列名叫 `BYS_TBB`，他只知道"增长了多少百分比"。产品必须完成这个映射。

#### 功能2.1: 预算类型 → 表名自动映射

**用户故事**: 作为预算分析员，我说"一般公共预算收入完成情况"，系统应该自动找到对应的表，而不需要我知道表名。

**用户输入 → 系统映射**:

| 用户说 | 系统 search_schema | 定位到的表 |
|--------|-------------------|-----------|
| "一般公共预算收入" | YBGGYS + 收入 | RDYS_LD_YSSC_YSZX_QSYBGGYSSRWC |
| "一般公共预算支出" | YBGGYS + 支出 | RDYS_LD_YSSC_YSZX_QSYBGGYSZCWC |
| "社保基金收入" | SHBXJJ + 收入 | RDYS_LD_YSZX_SHBXJJYSSRWC |
| "国有资本预算支出" | GYZBJY + 支出 | RDYS_LD_YSZX_GYZBJYYYZCWC |
| "政府性基金收支" | ZFXJJ | RDYS_LD_YSZX_ZFXJJ_SZQKJB |

**验收**: 20条口语化预算查询中，系统正确映射到目标表的准确率 ≥ 90%

#### 功能2.2: 按地区筛选

**用户故事**: 作为河北省的分析员，我最常查的就是本省的数据，偶尔需要对比各地市。

**详细需求**:

| 编号 | 用户输入 | 生成的 WHERE 条件 | 说明 |
|------|---------|------------------|------|
| 2.2.1 | "河北省" | `WHERE RG_NAME = '河北省'` | 精确匹配 |
| 2.2.2 | "石家庄" | `WHERE RG_NAME LIKE '%石家庄%'` | 模糊匹配 |
| 2.2.3 | "省本级" | `WHERE RG_NAME LIKE '%本级%'` | |
| 2.2.4 | "各地市" | `WHERE RG_NAME NOT LIKE '%本级%'` | 排除本级 |
| 2.2.5 | "石家庄和保定" | `WHERE RG_NAME IN ('石家庄市', '保定市')` | 多地区 |
| 2.2.6 | "河北各市排名" | 排除本级 + ORDER BY | 地区排名 |

#### 功能2.3: 按时间筛选

**用户故事**: 我想查某个月、某一年、或者最近几个月的数据。

**详细需求**:

| 编号 | 用户输入 | 生成的 WHERE 条件 | 说明 |
|------|---------|------------------|------|
| 2.3.1 | "2025年1月" | `WHERE YEAR_MONTH = '202501'` | |
| 2.3.2 | "今年" | 根据当前年份推断 | 智能年份推断 |
| 2.3.3 | "1月份" | `WHERE YEAR_MONTH LIKE '%01'` | 所有年份1月 |
| 2.3.4 | "2025年" | `WHERE YEAR_MONTH LIKE '2025%'` | 全年数据 |
| 2.3.5 | "最近3个月" | 计算出最近3个 YEAR_MONTH 值 | |

#### 功能2.4: 按项目名称筛选

**用户故事**: 我通常只看"合计"数，偶尔需要看税收收入或非税收入的明细。

**详细需求**:

| 编号 | 用户输入 | 生成的 WHERE 条件 |
|------|---------|------------------|
| 2.4.1 | "合计" | `WHERE XM_NAME LIKE '%合计%'` |
| 2.4.2 | "税收收入" | `WHERE XM_NAME LIKE '%税收%'` |
| 2.4.3 | "非税收入" | `WHERE XM_NAME LIKE '%非税%'` |
| 2.4.4 | "看合计和税收" | `WHERE XM_NAME IN ('...')` 或 OR 条件 |

#### 功能2.5: 同比对比查询（高频核心场景）

**用户故事**: 我最关心的是收入和去年同期比增长还是下降了，这是领导最常问的问题。

**业务背景**: 所有预算表都包含同比字段，这是用户最常看的指标。

| 编号 | 用户输入 | 系统行为 |
|------|---------|---------|
| 2.5.1 | "增长还是下降了" | 查询 BYS_JE, BYS_SNTYS, BYS_TBE, BYS_TBB |
| 2.5.2 | "同比下降的地区" | `WHERE BYS_TBE < 0` |
| 2.5.3 | "同比增长超过10%" | `WHERE BYS_TBB > 10` |
| 2.5.4 | "比去年多了多少" | 展示 BYS_TBE（同比增减额）和 BYS_TBB（同比增减率） |
| 2.5.5 | "和去年同期对比" | 展示 BYS_JE, BYS_SNTYS，对比两列 |

#### 功能2.6: 多条件组合查询

**用户故事**: 我可以一次性说出所有筛选条件，系统自动组合成完整查询。

**典型场景**:

| 用户输入 | 系统 SQL |
|---------|----------|
| "河北省2025年1月一般公共预算收入合计" | WHERE RG_NAME='河北省' AND YEAR_MONTH='202501' AND XM_NAME LIKE '%合计%' |
| "石家庄社保基金本月收入同比" | WHERE RG_NAME LIKE '%石家庄%' AND YEAR_MONTH=当月 AND XM_NAME LIKE '%合计%'，选同比列 |
| "各市2025年政府性基金累计收入，排除省本级" | WHERE YEAR_MONTH LIKE '2025%' AND RG_NAME NOT LIKE '%本级%'，选 BYLJS_JE |

#### 功能2.7: 智能列选择

**用户故事**: 不要给我 SELECT * 返回一大堆看不懂的列，只给我关心的几列就行。

| 用户关注点 | 应选的列 |
|-----------|---------|
| "完成情况" | RG_NAME, YEAR_MONTH, XM_NAME, YSS, BYS_JE, BYLJS_JE |
| "同比情况" | RG_NAME, YEAR_MONTH, XM_NAME, BYS_JE, BYS_SNTYS, BYS_TBE, BYS_TBB |
| "排名" | RG_NAME, 排序指标列 |
| "预算执行进度" | RG_NAME, YSS, BYS_JE, BYLJS_JE |

#### 阶段2 验收总标准

- 20条口语化预算查询，18条能正确生成 SQL（90% 准确率）
- 生成的 SQL 包含正确的 WHERE 条件、正确的列选择、正确的 ROWNUM 限制
- 所有 SQL 可以通过语法校验并在达梦数据库正确执行

### 阶段3: 汇总分析 — "能统计了" 📋

**产品价值**: 用户能做跨地区、跨时间的汇总统计、排名对比，回答"哪个最多/哪个最少/增长最快"这类决策性问题。

**用户故事**:
- 作为科室负责人，我想看各市的一般公共预算收入排名，以便了解全省情况
- 作为预算分析员，我想按月份汇总今年的收入，看趋势
- 作为领导，我想知道哪个地区税收收入最高，哪个最低

**核心功能**:

| 编号 | 功能 | 用户输入示例 | 对应 SQL |
|------|------|-------------|----------|
| 3.1 | 分组汇总 | "各市一般公共预算收入合计" | GROUP BY RG_NAME, SUM(BYS_JE) |
| 3.2 | 排名 | "哪个地市收入最高" | ORDER BY SUM(BYS_JE) DESC |
| 3.3 | 同比排名 | "收入同比增长最快的地区" | ORDER BY AVG(BYS_TBB) DESC |
| 3.4 | 时间趋势 | "今年各月收入变化趋势" | GROUP BY YEAR_MONTH ORDER BY YEAR_MONTH |
| 3.5 | 统计指标 | "全省平均/最高/最低收入" | AVG/MAX/MIN + GROUP BY |
| 3.6 | 汇总合计 | "2025年上半年总收入" | SUM WHERE YEAR_MONTH BETWEEN '202501' AND '202506' |
| 3.7 | 多维分组 | "各市各月的收入情况" | GROUP BY RG_NAME, YEAR_MONTH |

**验收标准**:
- "各市收入排名" → GROUP BY RG_NAME + ORDER BY SUM(BYS_JE) DESC
- "社保基金收入同比增长最快的前10个地区" → 多条件 + 排序 + ROWNUM
- "今年各月一般公共预算收入趋势" → GROUP BY YEAR_MONTH

## 质量基线（所有阶段通用）

1. **不编造**: 所有数据必须来自工具返回，禁止 LLM 编造表名/列名/数值
2. **不翻译**: 表名列名使用原始英文，可在旁边加中文说明但不替换
3. **语法正确**: 达梦用 ROWNUM（非 LIMIT），字符串用 ||（非 CONCAT）
4. **安全第一**: 只允许 SELECT，写操作需确认
5. **可读性**: SQL 格式化输出，结果用表格展示，Markdown 正确渲染
