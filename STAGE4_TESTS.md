# 阶段4 验收测试 — 跨表分析

> 所有问题均为预算分析人员真实口语。Agent 自动判断并行查询、UNION ALL 合并或 JOIN 关联策略。

## 一、跨预算对比 (8题)

| # | 用户问题 | 预期行为 |
|---|---------|---------|
| 4.1 | 一般公共预算收入和社保基金收入对比一下 | 分别查两表合计 → LLM 对比给出结论 |
| 4.2 | 政府性基金和国有资本预算，哪个支出规模更大 | 分别查两表合计 → 对比结论 |
| 4.3 | 一般公共预算支出和政府性基金支出差多少 | 分别查 → 计算差额和比例 |
| 4.4 | 社保基金和政府性基金收入，谁增长更快 | 分别查同比数据 → 对比增长率 |
| 4.5 | 四大预算收入排名，从高到低 | 分别查四类合计 → 汇总排名表 |
| 4.6 | 四大预算支出完成进度，哪个最好哪个最差 | 分别查预算数和累计 → 计算完成率排名 |
| 4.7 | 四大预算收入同比增长情况，给个全景 | 分别查同比 → 汇总对比表 |
| 4.8 | 今年哪个预算类型收入最高，哪个最低 | 查四类 → 找MAX/MIN → 排序 |

## 二、同构表合并 (5题)

| # | 用户问题 | 预期行为 |
|---|---------|---------|
| 4.9 | 四大预算各市收入总排名，合并到一起排 | UNION ALL → GROUP BY RG_NAME → SUM → ORDER BY |
| 4.10 | 各月四大预算的收入合计，看全年趋势 | UNION ALL → GROUP BY YEAR_MONTH → SUM → ORDER BY |
| 4.11 | 2025年上半年，四大预算支出合计是多少 | UNION ALL → WHERE BETWEEN → SUM |
| 4.12 | 河北省在四大预算里的收入加起来有多少 | UNION ALL → WHERE RG_NAME='河北省' → SUM |
| 4.13 | 四大预算合并后，各市累计收入排名前10 | UNION ALL → GROUP BY → SUM → ROWNUM <= 10 |

## 三、字典表关联 (5题)

| # | 用户问题 | 预期行为 |
|---|---------|---------|
| 4.14 | 按支出功能分类汇总一般公共预算支出 | JOIN RDYS_BAS_EXPFUNC ON EXP_FUNC_CODE → GROUP BY EXP_FUNC_NAME |
| 4.15 | 社保基金支出按功能分类，各类占比多少 | JOIN + GROUP BY → 计算占比 |
| 4.16 | 一般公共预算支出中，教育医疗社保分别花了多少 | JOIN → WHERE LIKE 过滤 → 汇总 |
| 4.17 | 按地区划分汇总一般公共预算收入 | JOIN RDYS_BAS_MOFDIV ON MOF_DIV_CODE → GROUP BY |
| 4.18 | 各支出分类在各地区的预算完成情况 | JOIN 两字典表 → 多维 GROUP BY |

## 四、表关系发现 (3题)

| # | 用户问题 | 预期行为 |

## Agent 自动策略清单

| 用户说的 | 策略 |
|---------|------|
| "XX和YY对比" | 分别查询 → LLM综合 |
| "四大预算合并" | UNION ALL → 统一聚合 |
| "按XX分类汇总" | JOIN字典表 ON编码列 → GROUP BY中文名 |
| "怎么关联" | find_relations → 建议JOIN条件 |

## 验收标准

- P0: 跨预算对比8题 ≥90% 正确对比
- P0: 同构表合并5题 ≥80% 正确UNION ALL
- P0: 字典表关联5题 ≥80% 正确JOIN
- P1: 表关系发现3题 使用find_relations
- P1: 组合场景5题 综合运用多策略
- P2: 边界测试4题 降级建议合理
