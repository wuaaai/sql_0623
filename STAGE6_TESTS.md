# 阶段6 验收测试 — 会话记忆

> 跨会话上下文持久化、查询历史、个性化默认值。15题，全部口语化。

## 一、会话恢复（5题）

| # | 用户问题 | 预期行为 |
|---|---------|---------|
| 6.1 | 昨天查了河北省一般公共预算收入，今天打开浏览器说"那社保的呢" | 自动恢复上下文（河北省+一般公共预算），只替换预算类型为社保基金 |
| 6.2 | 上次分析了河北省一般公共预算4月数据，这次说"看看石家庄的" | 复用预算类型和时间，地区从河北省替换为石家庄市 |
| 6.3 | 上次看的是河北省4月数据，这次说"看看5月的" | 复用地区和预算类型，时间更新为5月 |
| 6.4 | 先查河北省一般公共预算，切换到石家庄社保基金，然后说"那支出情况呢" | 基于最新上下文（石家庄+社保）继续追问，不回到最初的河北省 |
| 6.5 | 点击"开始新对话"后，问"收入多少" | 上下文完全清空，Agent提示用户指定预算类型和地区 |

## 二、查询历史（4题）

| # | 用户问题 | 预期行为 |
|---|---------|---------|
| 6.6 | 昨天查了哪些问题，帮我看看 | 展示历史查询列表（问题+SQL+时间），按时间倒序 |
| 6.7 | 上周那条"各地市收入排名"再跑一遍 | 点击历史记录，重新执行相同SQL，展示最新结果 |
| 6.8 | 之前查社保基金支出那次，SQL是怎么写的 | 从历史记录中找到对应条目，展示完整SQL |
| 6.9 | 历史里各地市收入排名，把前10改成前5再查一次 | 复制历史上下文和SQL，修改TOP N后重新执行 |

## 三、个性化默认值（3题）

| # | 用户问题 | 预期行为 |
|---|---------|---------|
| 6.10 | 我以后都查河北省的数据，帮我设成默认 | 保存默认地区=河北省到localStorage，后续查询自动带入 |
| 6.11 | 默认预算设成一般公共预算，以后只说"收入多少"就查这个 | 保存默认预算类型，省略时自动补充 |
| 6.12 | 算了，把默认值都清掉吧 | 清除localStorage默认值，后续不再自动带入 |

## 四、边界（3题）

| # | 用户问题 | 预期行为 |
|---|---------|---------|
| 6.13 | 第一次用这个系统，上来就问"河北省收入多少" | 无历史会话，正常初始化空会话，不报错 |
| 6.14 | 会话文件JSON格式坏了 | 检测损坏→提示"会话数据异常，已创建新会话"，不崩溃 |
| 6.15 | 查了上千条记录了，历史列表会不会卡 | 分页或只展示最近50条，不卡顿 |

## 验收标准

- P0: 会话恢复 5题 ≥80%
- P0: 查询历史 4题 可查看/重执行
- P1: 个性化默认值 3题
- P1: 边界 3题 降级不崩溃
