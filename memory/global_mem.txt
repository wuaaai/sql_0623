# 全局记忆 (L2)

## 达梦语法要点
- 限制行数: WHERE ROWNUM <= N (不是 LIMIT)
- 字符串拼接: 使用 || (不是 CONCAT)
- ORDER BY + ROWNUM: 必须嵌套子查询 (ROWNUM先于ORDER BY执行)

## 数据库概况
- Schema: RDYS_PUBLIC_TBS，共47张表
- 预算类型和列名模式: 参见 memory/business_rules.json

## 重要: 市县数据查XM_NAME而非RG_NAME
RG_NAME 固定为"河北省"或"河北省本级"——是省级汇总。
市县数据在 XM_NAME 列中，例如"衡水市""赞皇县""石家庄市本级"。
查某市县: WHERE XM_NAME LIKE '%衡水市%'
查各区县: WHERE XM_NAME LIKE '%县%'

## 排名查询
- 查"各市排名"直接用预算表: SELECT RG_NAME, BYS_JE FROM table WHERE ... ORDER BY BYS_JE DESC
- 不需要 GROUP BY，每行已是一个地区的数据，直接 ORDER BY + ROWNUM 即可

## 窗口函数语法（达梦支持）
- SUM(col) OVER(): 窗口合计（占比分母）
- LAG(col, n) OVER (PARTITION BY x ORDER BY y): 前n行值（环比）
- AVG/STDDEV OVER(): 窗口均值/标准差（异常检测）

## 异常检测阈值
- threshold=2.0: 常规(95%置信区间外)
- threshold=3.0: 极端(99.7%外)，用于重大发现汇报

## 跨表关联知识
### 字典表关联键
- RDYS_BAS_EXPFUNC(支出分类): EXP_FUNC_CODE → 执行表
- RDYS_BAS_MOFDIV(地区划分): MOF_DIV_CODE → 执行表
- 使用 find_relations 工具发现两表的关联键
### 跨表查询策略
- 对比 → 分别查两张表 + LLM综合
- 合并排名 → union_query 或 run_sql(UNION ALL)
- 字典关联 → find_relations → run_sql(JOIN)
