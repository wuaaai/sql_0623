# 进度描述规范

## 规则

1. 每个执行步骤必须输出一行进度描述（20字以内）
2. 用"正在..."开头，完成后用"已..."开头
3. 不超过5行进度，合并同类步骤
4. 禁止输出内部技术细节（如表名、列名、匹配状态）
5. 禁止说"很好！""匹配到历史模式""确认RG_NAME"等技术过程

---

## 进度模板

### 查询启动
```
正在分析您的问题...
```

### 表定位（合并search_schema/describe_table）
```
正在查找相关数据表...
已定位数据表，正在查询...
```

### 简单数据查询（1步SQL）
```
正在查询数据...
已获取结果，正在整理...
```

### 聚合/排名查询（GROUP BY）
```
正在汇总统计数据...
已生成排名，正在整理...
```

### 同比对比查询
```
正在查询本年及同期数据...
已获取对比结果，正在分析...
```

### 多步复杂查询（RAG+SQS）
```
正在检索相关政策文档...
已找到政策依据，正在查询执行数据...
```

### 跨表对比
```
正在分别查询各项数据...
已获取对比数据，正在汇总...
```

### 错误重试
```
正在优化查询条件...
已重新查询，正在整理结果...
```

### 查询完成
```
查询完成，结果如下：
```

## 示例对比

### 之前（内部技术细节过多）
```
我来查一下相关表的结构和具体数据。
很好！已经有完全匹配的历史模式。
不过我需要确认"全省"对应的RG_NAME，先看一下数据。
好的，"全省"数据的RG_NAME是"河北省本级"，现在查询各月执行数。
```

```
正在分析您的问题...
正在查询2025年1-5月全省国内增值税各月执行数据...
已获取结果，正在整理...
查询完成，结果如下：
```

## 使用方式

system_prompt 注入时附带此文件内容。
当Agent开始执行工具调用时，在回复正文开头输出对应阶段的进度描述。
