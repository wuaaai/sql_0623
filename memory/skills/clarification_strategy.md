# 追问策略设计

## 核心原则
- 最多连续追问2次，第3轮必须自动执行
- 一次问完所有缺失条件，不逐个追问
- 自动执行时透明告知用户用了什么默认值

## 条件优先级

| 级别 | 条件 | 处理策略 |
|------|------|---------|
| P0 | 预算类型、查询意图 | 必须确认。可推断("社保"→SHBXJJ)，但裸"预算"必须问 |
| P1 | 地区、时间、项目 | 可默认：地区→全部、时间→最新、项目→合计 |
| P2 | 行数限制、排序 | 永不问。默认20行，自然顺序 |

## 两轮降级路径

### 第1轮追问（clarify_count=0）
- 问 P0 缺口（预算类型不可推断时、查询意图模糊时）
- P0 已清晰则批量问 P1 缺口
- 话术示例："您关注哪类预算？一般公共预算、社保基金、国有资本，还是政府性基金？"

### 第2轮追问（clarify_count=1）
- 确认剩余 P1 缺口
- 话术示例："好的，[已确认的条件]。剩下两个确认：哪个地区？哪个月份？不指定的话默认查全部地区最新数据。"

### 第3轮自动执行（clarify_count>=2）
- 系统注入强制指令，不再追问
- 所有缺失条件自动填默认值
- 查询结果前加告知："已为您查询[地区范围][预算类型][时间]数据"

## 默认值策略

| 条件 | 默认值 | SQL体现 |
|------|--------|---------|
| 时间 | resolve_time("") 最新数据 | WHERE YEAR_MONTH = '最新' |
| 地区 | 不筛选 | 无 RG_NAME 条件 |
| 项目 | 合计 | WHERE XM_NAME LIKE '%合计%' |
| 预算类型 | 最后兜底：一般公共预算 | 告知用户"默认查询一般公共预算" |
| 查询意图 | 完成情况 | 选 BYS_JE, BYLJS_JE, YSS |

## 追问计数规则

- Agent的回答包含"？"或"?" → 计数+1
- Agent给出了查询结果（不包含"？"）→ 计数归零
- 用户开启新话题（换了完全不同的预算类型或问题类型）→ 计数归零

## 与 interaction_patterns.md 的关系

本文件定义追问的**频率和降级策略**。
interaction_patterns.md 定义追问的**话术和上下文记忆**。
两者互补，本文件的规则优先级更高。
