# 会话记忆策略

## 核心机制

服务端自动保存会话上下文到 sessions/current.json，每次查询时注入 system_prompt。
对 LLM 透明——LLM 看到上下文后自然推断复用。

## 复用策略

| 场景 | 行为 |
|------|------|
| 用户说"社保的呢"（省略预算类型） | 复用上次的预算类型，只切换 |
| 用户说"石家庄的呢"（省略预算+时间） | 复用上次预算和时间，切换地区 |
| 用户说"开始新对话" | 清空上下文 |
| 用户明确说"河北省一般公共预算" | 用户明确说了，不需要复用 |

## 上下文记忆维度
- 预算类型: 一般公共预算/社保/国资/政府性基金
- 地区: RG_NAME 值
- 时间: YEAR_MONTH 值
- 上次查询的表名和SQL

## 优先级
用户明确说的 > 会话上下文 > 用户默认值 > 系统默认值
