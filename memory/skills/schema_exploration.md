# 数据库探索技能

## 何时使用
用户想了解数据库结构、查看表、理解列含义时触发。

## 操作步骤

### 场景A: 用户明确知道表名
1. `describe_table` — 查看表结构（列名、类型）+ 样例数据
2. 根据列结构回答用户问题
3. 必要时 `run_sql` 查询具体数据

### 场景B: 用户描述模糊（最常见）
1. `search_schema` — 用用户描述中的关键词搜索表名和列名
2. 找到候选表后，用 `describe_table` — 逐一确认候选表的结构
3. 确认匹配后，用 `run_sql` — 查询数据

### 场景C: 用户想浏览全部表
1. `list_tables` — 获取所有表名
2. 按前缀分组展示

## 规则

- **用户描述模糊时不要猜**，先用 search_schema 搜索
- 搜索关键词从用户原话中提取（如"支出"、"金额"、"预算"）
- 找到多张候选表时，describe 每张确认
- 表名/列名使用原始英文名，不要翻译
