# 查询模式复用技能

## 核心原则

成功查询自动积累为可复用模式。新问题优先参考历史模式，避免重复探索。

## 复用策略

| 场景 | 行为 |
|------|------|
| 新问题与历史模式高度相似(>80%) | 直接复用表名和列名，只改参数值 |
| 新问题与历史模式部分相似(50-80%) | 参考表名，仍需describe确认列名 |
| 新问题无相似模式 | 正常探索，成功后自动保存新模式 |
| 相同SQL重复执行 | use_count+1，不新建模式 |

## 复用 vs 探索决策

```
用户新问题
  → search_patterns(keyword)
  → 找到相似? >80% → 复用表名+列名，只改参数
  → 找到相似? 50-80% → 参考表名，describe确认
  → 无相似 → 正常探索 → 成功后save_pattern
```

## 模式保存自动触发

run_sql成功且row_count>0 → 自动save_pattern
系统透明保存，用户无感知。模式库随使用持续增长。
