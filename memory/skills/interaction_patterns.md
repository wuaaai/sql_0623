# 交互对话技能 — 上下文推断与追问

## 核心原则

**能推断就不问。能默认就不问。追问是最后手段。**

## 上下文自动推断（优先级最高）

用户说的每一句话都可能省略信息，Agent 必须先尝试从上下文补全，不能用默认值时才考虑追问。

### 上下文记忆维度

| 维度 | 记忆内容 | 复用规则 |
|------|---------|---------|
| 预算类型 | 上一轮查过的预算(一般公共/社保/国资/政府性基金) | 用户追问同类问题时复用 |
| 地区 | 上一轮筛选的地区 | 用户追问同类问题时复用 |
| 时间 | 上一轮用的 YEAR_MONTH | 用户追问同类问题时复用，时间词变化时更新 |
| 项目 | 上一轮筛选的 XM_NAME | 默认"合计"，用户明确切换时更新 |

### 上下文推断示例

```
第1轮: "河北省一般公共预算收入完成得怎么样"
  → 记忆: 预算=一般公共预算, 地区=河北, 时间=最新

第2轮: "社保的呢"  
  → 推断: 地区=河北(复用), 时间=最新(复用), 预算=社保(用户说了)
  → 不追问，直接执行 ✅

第3轮: "石家庄的呢"
  → 推断: 预算=社保(复用第2轮), 时间=最新(复用), 地区=石家庄(用户说了)
  → 不追问，直接执行 ✅

第4轮: "看看同比"
  → 推断: 预算=社保(复用), 地区=石家庄(复用), 时间=最新(复用), 意图=同比
  → 不追问，直接执行 ✅
```

### 上下文重置时机
- 用户明确否定："不是/不对/换一个"
- 用户完全换了话题："数据库有哪些表"
- 查询成功后超过1轮没有复用

## 自动默认值策略（推断不出时）

| 缺失条件 | 默认值 | 是否追问 |
|---------|--------|---------|
| 时间 | resolve_time("") 数据库最新 | 不追问 |
| 地区 | 不筛选（全部地区） | 不追问 |
| 项目 | "合计" | 不追问 |
| 预算类型 | 上文复用 > 默认一般公共预算 | **仅此项可追问** |

**只有当预算类型完全无法从上下文推断且用户描述过于模糊时，才启动追问。**

## 追问话术（极少使用）

### 追问1（仅预算类型不明时）
用户: "收入完成得怎么样"（无上文、无预算类型）
→ Agent: "您关注哪类预算？一般公共预算、社保基金、国有资本，还是政府性基金？"
→ 追问后用户补全 → 直接执行

### 追问2（追问1后仍不明确时）
用户: "都看看" 或 "随便"
→ Agent: 不再追问，默认一般公共预算 + 全部地区 + 最新时间 + 合计
→ 告知: "已为您查询一般公共预算收入最新数据。"

## 结果为空时的处理

用户: "火星省的收入"
→ `WHERE RG_NAME LIKE '%火星%'` → 0行
→ 回复: "未找到'火星'相关的地区。已展示全部地区数据供参考。" → 展示全部地区结果

## 跨阶段降级

用户: "一般公共预算和政府性基金哪个增长快"
→ 当前阶段不支持跨表对比
→ 回复: "我分别帮您查看：1) 一般公共预算收入 2) 政府性基金收入"
→ 直接执行查询1，然后问"需要继续看政府性基金吗？"
