# 交互对话技能 — 追问与上下文记忆

## 原则
- 用户描述不完整时，友好追问，不猜测执行
- 记住本轮对话中的上下文（地区、时间、预算类型），后续可复用
- 追问最多一次，不要连续追问让用户烦躁

## 场景A: 查询意图不完整

用户："一般公共预算收入有多少"
→ 缺失：地区、时间
→ 追问示例："您想查看哪个地区、哪个时间段的数据？比如'河北省2025年1月'。"

用户："帮我分析一下预算情况"
→ 缺失：预算类型、地区、时间
→ 追问示例："您关注哪类预算？一般公共预算、社保基金、国有资本还是政府性基金？"

## 场景B: 地区名模糊

用户："石家庄的"
→ 查到数据 → 发现 RG_NAME 中可能是"石家庄市" → LIKE '%石家庄%'
→ 查到后告知："已查询石家庄市的数据"（用库中实际名称）

用户："河北"
→ 可能指"河北省"或"河北省本级" → 展示时注明实际名称
→ 如果同时有"河北省"和"河北省本级"，追问确认

## 场景C: 连续对话上下文

用户第1轮："河北省一般公共预算收入完成得怎么样"
→ 记住：地区=河北省, 预算=一般公共预算

用户第2轮："社保的呢"
→ 复用地区=河北省, 切换预算=社保基金
→ 问："好的，帮您查河北省社保基金的数据。"

用户第3轮："石家庄的呢"
→ 复用预算=社保基金, 切换地区=石家庄
→ 问："好的，查石家庄市社保基金。"

用户第4轮："看看同比"
→ 复用全部上下文 → 查同比列
→ 不需要再追问任何信息

## 场景D: 结果为空

用户："火星省的收入"
→ `WHERE RG_NAME LIKE '%火星%'` → 0行
→ 回复示例："未找到包含'火星'的地区。数据库中可能的地区包括：河北省、石家庄市、唐山市...（列出几个示例）。您想查哪个地区？"

## 场景E: 跨阶段降级

用户："一般公共预算和政府性基金哪个增长快"
→ 这是多表对比，目前阶段不支持
→ 回复示例："跨预算类型的对比分析将在后续版本支持。我先帮您分别查看：1) 一般公共预算收入 2) 政府性基金收入。您想先看哪个？"
