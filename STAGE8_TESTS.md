# 阶段8 验收测试 — RAG + Text-to-SQL 融合

> 知识库文档: 2019-2026年河北省预算解读、邯郸市预算/计划解读、十五五规划纲要
> 测试目标: 验证 LLM 正确路由到 SQL 工具或 RAG 工具，混合场景综合回答

---

## 一、纯 RAG 知识问答（8题）

用户问政策/解读类问题，应调 `rag_search`。

| # | 用户问题 | 预期工具 | 预期回答要点 |
|---|---------|---------|------------|
| R1 | 2026年河北省预算编制有什么新政策 | rag_search | 基于预算解读文档回答，含具体政策条款 |
| R2 | 一般公共预算和政府性基金预算有什么区别 | rag_search | 解释四本预算的定义和区别 |
| R3 | 2026年预算解读中提到的重点支出领域有哪些 | rag_search | 列出教育/科技/社保等重点领域 |
| R4 | 邯郸市十五五规划纲要的主要内容是什么 | rag_search | 基于邯郸市规划文档回答 |
| R5 | 中央对地方转移支付的最新政策要求 | rag_search | 基于政策文档回答转移支付规定 |
| R6 | 2025年和2026年预算编制要求有什么变化 | rag_search | 对比两年解读文档 |
| R7 | 预算绩效管理改革的进展 | rag_search | 基于文档回答改革方向和进展 |
| R8 | 河北省2024年预算执行情况和2025年草案 | rag_search | 基于预算执行报告文档 |

---

## 二、纯 SQL 数据查询（6题）

用户问数据类问题，应调 `run_sql` 或 `run_aggregation`。**RAG集成后原SQL功能不受影响。**

| # | 用户问题 | 预期工具链 | 预期结果 |
|---|---------|-----------|---------|
| S1 | 河北省2026年一般公共预算收入完成多少 | search_schema→describe→run_sql | 返回具体金额数据 |
| S2 | 各市一般公共预算收入排名前5 | search_schema→describe→run_aggregation | 排名表+金额 |
| S3 | 社保基金收入同比增长最快的地区 | search_schema→describe→run_aggregation | 同比排名 |
| S4 | 2025年全年政府性基金收入汇总 | resolve_time→search_schema→run_aggregation | 汇总金额 |
| S5 | 衡水市今年各月收入趋势 | search_schema→describe→run_sql | 逐月趋势表 |
| S6 | 各地市收入占全省比例 | search_schema→calc_ratio | 占比百分比 |

## 三、混合查询（6题）

用户问题同时涉及知识库和数据库，应先 `rag_search` 理解背景，再 `run_sql` 查数据。

| # | 用户问题 | 预期流程 | 预期结果 |
|---|---------|---------|---------|
| H1 | 预算解读中提到的新增专项转移支付，实际执行了多少 | rag_search(专项转移支付)→提取关键词→search_schema→run_sql | 先解释政策，再展示执行数据 |
| H2 | 2026年教育支出的预算安排是多少，实际完成得怎么样 | rag_search(教育支出政策)→理解预算安排→run_sql查实际数据 | 政策安排 vs 实际完成对比 |
| H3 | 邯郸市十五五规划中提到的重点项目，资金到位情况如何 | rag_search(邯郸规划)→提取重点项目名→search_schema→run_sql | 项目清单+资金数据 |
| H4 | 今年省本级一般公共预算支出比去年增加了吗，政策依据是什么 | run_sql查同比→rag_search查政策依据 | 数据变化+政策解释 |
| H5 | 解读里提到要加大科技投入，看看今年科技支出实际增长了没有 | rag_search(科技投入政策)→search_schema→describe→run_sql | 政策要求+实际增长数据 |
| H6 | 预算执行报告里说的进度偏慢的地市，看看他们的实际完成率 | rag_search(执行进度)→search_schema→run_aggregation(各市完成率) | 政策点评+具体数据 |

## 四、路由准确性测试（5题）

验证 LLM 选择正确工具链的判断力。

| # | 用户问题 | 正确工具 | 常见错误 |
|---|---------|---------|---------|
| A1 | 帮我分析一下预算情况 | 追问(模糊) | 不应直接调任一工具 |
| A2 | 收入支出平衡吗 | run_sql/run_aggregation | 不应调 rag_search |
| A3 | 新的预算管理制度改革方向 | rag_search | 不应调 run_sql |
| A4 | 河北省一般公共预算收入和去年比 | run_sql(同比列) | 不应调 rag_search |
| A5 | 怎么理解财政转移支付制度 | rag_search | 不应调 run_sql |

## 五、边界与异常（5题）

| # | 用户问题 | 场景 | 预期行为 |
|---|---------|------|---------|
| B1 | 2028年预算展望 | 知识库无此年份文档 | rag_search 返回空→告知"暂无2028年相关文档" |
| B2 | RAG服务不可用时的知识问题 | Vastbase连接断开 | rag_search 返回错误→告知"知识库暂不可用，建议换个问题" |
| B3 | 一口气问10个不同预算类型的收入数据 | 数据库查询正常 | 正常执行SQL，不受RAG影响 |
| B4 | 同时问5个政策问题 | 知识库检索 | 逐条检索，不超时 |
| B5 | SQL查询结果为空 | 数据库无匹配数据 | "结果为空"提示，不让LLM编造 |
